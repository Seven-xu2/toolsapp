# 运行依赖说明

## 一、当前版本的运行模式

当前项目已支持两种使用方式：

- **游客模式**：不注册、不登录，直接使用本地工具
- **登录模式**：注册/登录后，使用云同步与账号相关功能

因此，当前版本已经不再是“必须先登录才能使用”的形态。

## 二、哪些功能不依赖后端

以下功能在游客模式下可直接使用，不需要后端服务：

- 计算器
- 单位转换
- 手电筒
- 二维码生成
- 时间戳转换
- 文本工具

这些功能的核心计算或本地能力调用都在前端完成。

## 三、哪些功能依赖后端

以下功能仍然需要后端服务和数据库支持：

- 用户注册
- 用户登录
- 收藏/取消收藏
- 收藏列表
- 使用历史写入、查询、删除、清空
- 个人中心资料查询
- 个人中心统计信息查询
- 修改昵称
- 后续新增的云端型工具

对应依赖包括：

- Spring Boot 后端接口
- JWT 鉴权
- MySQL 数据持久化

## 四、下载 APK 后是否还需要部署后端

### 情况 1：只想使用本地工具

**不一定需要。**

如果你只是想让用户安装 APK 后使用本地工具，那么现在可以直接通过“游客进入”使用，不依赖后端。

### 情况 2：想使用账号、收藏、历史、个人中心

**需要。**

如果你希望用户能注册登录并使用云同步能力，就仍然需要部署后端服务和 MySQL 数据库。

## 五、当前版本在没有后端时的表现

如果后端没有部署或无法连接：

- 用户仍可通过游客模式进入首页
- 首页会自动使用前端内置的本地工具目录
- 本地工具可正常打开并使用
- 点击收藏、历史、个人中心等云端功能时，会提示去注册/登录
- 如果未来新增了“需要后端支持”的工具，也可以通过工具元数据进行拦截提示

## 六、后续扩展方式

为了兼容未来新增工具，前端已增加工具元数据字段：

- `requiresBackend`
- `route`

建议后续新增工具时：

- 本地工具：`requiresBackend: false`
- 云端工具：`requiresBackend: true`

这样在游客模式下：

- 本地工具可直接进入
- 云端工具会统一弹出“需要注册/登录”的提示

## 七、推荐使用建议

### 如果你暂时不准备部署后端

当前推荐方式：

- 直接打包 APK
- 用户通过“游客进入”使用本地工具
- 先不开放云同步能力的正式交付承诺

### 如果后续你准备部署后端

可以继续启用：

- 注册登录
- 收藏同步
- 历史同步
- 个人中心
- 后续云端型工具

你可以直接把下面这段话发给最终用户：
